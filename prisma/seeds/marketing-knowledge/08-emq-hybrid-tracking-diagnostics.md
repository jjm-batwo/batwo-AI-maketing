# EMQ (이벤트 매칭 품질) 및 하이브리드 트래킹 진단 가이드

## 개요
Meta 광고 플랫폼에서 **EMQ(Event Match Quality, 이벤트 매칭 품질)**는 1점(낮음)부터 10점(높음)까지로 평가되며, 웹사이트를 방문한 고객의 데이터가 Meta 사용자 계정과 얼마나 잘 일치하는지를 나타냅니다. CAPI(전환 API)와 브라우저 픽셀을 동시에 활용하는 하이브리드 트래킹 환경에서 높은 매칭률은 ROAS를 직접적으로 상승시키는 핵심 요인입니다.

## EMQ 점수 진단 및 해결 방안

### 1. 매칭률 임계값
- **우수 (8.0 ~ 10.0)**: 이상적인 데이터 흐름. CAPI와 픽셀 데이터가 정상적으로 수집/매칭되며 리타겟팅과 Advantage+ 캠페인 성과가 극대화됩니다.
- **관찰/경고 (4.0 ~ 7.9)**: 일부 추가 데이터 매칭이 가능합니다. 이메일이나 전화번호와 같은 PII 파라미터가 누락되었는지 확인해야 합니다.
- **위험 (< 4.0)**: 광고 성과가 심각하게 저하될 수 있습니다. CAPI 설정 오류, 잘못된 데이터 해싱, 또는 사용자 식별 정보 누락이 의심됩니다.

### 2. 매칭률 하락 5대 주요 원인
1. **userData 누락 (`em`, `ph`)**: 결제나 회원가입 시 이메일(`em`)과 전화번호(`ph`) 해싱 데이터가 전송되지 않음
2. **fbc / fbp 쿠키 미수집**: `_fbp` (브라우저 ID) 및 `_fbc` (클릭 ID) 쿠키 파라미터가 백엔드로 전달되지 않음
3. **잘못된 데이터 해싱 방법**: 서버 측에서 PII 데이터를 소문자로 변환하고 공백 제거 후 SHA-256 해싱해야 함 (현재 시스템은 자동 처리됨)
4. **Client IP 누락 (`client_ip_address`)**: CAPI로 서버에서 전송 시 사용자의 실제 IP가 아닌 백엔드 서버 IP가 전달됨 (Proxy, Cloudflare 환경 등에서 유의)
5. **Client User Agent 누락 (`client_user_agent`)**: 브라우저 환경 정보 누락으로 사용자 기기 유추 실패

## 하이브리드 트래킹 작동 및 중복 제거(Deduplication)

CAPI(서버 전송)와 픽셀(브라우저 전송)을 함께 사용하는 경우, Meta는 중복된 전환 이벤트를 식별하고 제거하여 데이터 신뢰성을 보장합니다.

### 어떻게 중복이 제거되는가?
Batwo 시스템을 통해 전송되는 브라우저 이벤트와 서버(CAPI) 이벤트가 동시에 Meta로 전달되었을 때, Meta의 수신 서버는 **항상 브라우저 이벤트를 우선 처리**합니다.

1. **`event_name` 필수 일치**: 두 이벤트 모두 정확히 같은 이름(`Purchase`, `AddToCart` 등)이어야 합니다.
2. **`event_id` 필수 일치**: **핵심!** 동일한 액션(예: 하나의 장바구니 담기)에 대해 브라우저와 서버는 정확히 같은 `event_id` 파라미터 값을 전송해야 합니다.
   - 예: 브라우저에서 `event_id: "order_12345"`를 보냈을 때 CAPI 역시 `event_id: "order_12345"`여야 함.
3. 데이터는 48시간 이내에 전송되어야 중복이 제거될 수 있습니다.

### 중복 제거 문제 진단 (하이브리드 대시보드 참고)
- **CAPI 이벤트 수가 브라우저의 10% 미만인 경우**: 
  - 서버 측 통신 장애나 CAPI 크론 배치가 실패하고 있을 수 있습니다. 배치 설정 및 백엔드 로그 확인이 필요합니다.
- **Meta 통계상 이벤트 수가 시스템 실제 지표 대비 2배 가까운 경우**:
  - `event_id`가 일치하지 않아 중복 제거에 실패한 상태입니다. 클라이언트 사이드와 서버 사이드에서 동일한 UUID나 주문/세션 ID를 동기화하여 사용하는지 점검하세요.

## CAPI 상태 추적
현재 픽셀 대시보드의 하이브리드 요약 카드에서 CAPI 전송의 다음과 같은 통계를 확인할 수 있습니다.
- **전송 실패 (FAILED)**: 토큰 만료 또는 페이로드 오류. Meta Graph API 버전(v25.0)의 제약 변경일 수 있습니다.
- **시효 만료 (EXPIRED)**: 이벤트 수집 후 전송 시점에 서버 문제로 7일이 초과된 데이터. Meta는 수집 후 7일 이전의 데이터 이력 배치를 허용하지 않아 자동 파기합니다.
