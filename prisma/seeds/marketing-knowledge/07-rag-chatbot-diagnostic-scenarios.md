# RAG 챗봇 진단 시나리오: 프롬프트 엔지니어링 매핑 구조 및 자동 대응 로직

SaaS 내부에 구축될 RAG 챗봇의 핵심은 사용자의 **단편적이고 비전문적인 질문을 알고리즘 관점의 깊이 있는 진단으로 치환**하고, 즉각적으로 시스템에 적용 가능한 실행 방안을 생성하는 데 있다.

---

## 시나리오 1: 학습 단계(Learning Phase) 실패 및 예산 소진 정체

### 사용자 발화
> "어제 야심 차게 캠페인을 세팅했는데 예산이 전혀 소진되지 않고 있습니다. 문제가 무엇인가요?"

### RAG 검색 및 분석 로직
1. 사용자의 광고 계정 API를 스캔하여 예산 구조와 입찰 전략 파악
2. 검색된 지식 베이스(Andromeda 렌더링 방식, Advantage+ 학습 단계 조건, 가치 기반 입찰의 제약)와 대조

### SaaS 챗봇 자동 답변 예시
> "현재 해당 광고 세트에 설정된 일일 예산이 $20 수준으로 파편화되어 있어, 2026년 알고리즘이 요구하는 일주일 내 50회 전환 데이터 수집 기준을 충족하지 못하고 있습니다. 또한 설정하신 '수동 입찰가 제한(Cost Cap)'이 메타의 Lattice 엔진이 예측한 실제 경매 단가보다 낮아 시스템이 아예 경매 진입을 차단한 상태입니다.
> 
> 즉각적인 문제 해결을 위해 파편화된 광고 세트를 하나로 통합하여 일일 예산을 $50 이상으로 집중시키고, 입찰 옵션을 '최대 볼륨(Highest Volume)'으로 변경하여 3일간 관망하며 알고리즘의 학습 단계 통과를 유도하시기를 강력히 권장합니다. 자동 수정을 진행해 드릴까요?"

## 시나리오 2: 광고 피로도에 따른 ROAS 하락 및 CPM 급등

### 사용자 발화
> "지난달까지는 성과가 매우 훌륭했던 영상 소재인데, 이번 주부터 전환이 급감하고 단가가 2배 넘게 뛰었습니다."

### RAG 검색 및 분석 로직
1. API를 통해 해당 소재의 빈도수(Frequency) 및 도달률 지표 수집
2. 컴퓨터 비전 모델로 소재 간 시각적 유사도(Creative Similarity) 점수 추출
3. Andromeda의 노출 페널티 로직과 결합 분석

### SaaS 챗봇 자동 답변 예시
> "데이터 분석 결과, 해당 캠페인의 소재가 동일 사용자에게 평균 3.5회 이상 반복 노출되며 'Creative Fatigue(광고 피로도)' 임계치를 초과한 것으로 확인되었습니다. 이로 인해 Andromeda 엔진이 CPA를 상승시키는 구조적 페널티를 부과하고 있습니다.
> 
> 기존 영상을 그대로 복제하여 자막만 수정하지 마시고, 영상의 썸네일 색감, 초반 3초의 시각적 훅(Hook), 영상의 포맷 자체(예: 화려한 스튜디오 컷에서 일상적인 고객 리뷰 스타일로 변경)를 완전히 뒤엎은 2~3개의 새로운 'Entity ID' 소재를 즉시 시스템에 투입하십시오. 자사의 생성형 AI 기능으로 즉시 대체 영상을 제작해 드릴 수 있습니다."

## 시나리오 3: 잠재 고객(Lead Ads) 품질 저하 (2025년 10월 정책 업데이트)

### 사용자 발화
> "리드 광고로 수집된 DB(전화번호, 이메일)에 연락해보면 부재중이거나 관심이 없다는 허수 고객이 너무 많습니다."

### RAG 검색 및 분석 로직
1. 2025년 10월에 발효된 Meta Lead Ads의 데이터 투명성 및 규제 업데이트 가이드라인 검색
2. 현재 폼 양식의 마찰(Friction) 수준 분석

### SaaS 챗봇 자동 답변 예시
> "2025년 10월 메타는 잠재 고객 확보 광고의 데이터 수집과 동의 과정에 관한 엄격한 투명성 정책 업데이트를 단행했습니다. 단순히 양만 늘리기 위해 사용자의 프로필 정보를 그대로 끌어오는 자동 채우기(Autofill) 방식의 폼은 2026년 현재 허수 데이터를 양산하는 주범입니다.
> 
> 품질을 높이기 위해 폼 양식 내에 '비즈니스 전용 이메일' 필수 입력 조건을 걸거나, SMS 인증 프로세스, 혹은 사용자가 직접 타이핑해야 하는 주관식 질문(예: '도입을 희망하는 시기는 언제입니까?')을 의도적으로 추가하여 제출 과정의 마찰(Friction)을 높이십시오.
>
> 또한, 메타의 **'가치 규칙(Value Rules)'**을 시스템 설정에서 적용하여 특정 고가치 기기(예: iOS/Android)나 구매 전환율이 높은 지역의 입찰가를 시스템이 자동으로 올리도록(최대 1,000%까지 조정 가능) 학습시키십시오. 이를 통해 전환 '수량'이 아닌 '가치'를 기준으로 최적화할 수 있습니다."

## 시나리오 4: SaaS 내 캠페인 생성 오류 — API 마이그레이션 진단

### 사용자 발화
> "대시보드에서 새 캠페인을 만들려고 하는데 오류가 발생하고 생성이 안 됩니다."

### RAG 검색 및 분석 로직
1. 메타 마케팅 API(MAPI) v25.0 강제 적용 규정 검색
2. 현재 SaaS 백엔드가 사용 중인 API 버전 및 캠페인 생성 엔드포인트 확인
3. 레거시 ASC/AAC 방식의 구버전 코드 사용 여부 진단

### SaaS 챗봇 자동 답변 예시
> "캠페인 생성 오류의 원인으로 API 마이그레이션 이슈가 감지되었습니다. 메타 마케팅 API v25.0 정책에 따라, 구버전 방식의 레거시 Advantage+ 쇼핑 캠페인(ASC) 및 앱 캠페인(AAC)을 생성하거나 수정하는 기능이 원천 금지되었으며, 이 규정은 2026년 5월 19일까지 모든 하위 API 버전에 강제 적용됩니다.
>
> 현재 시스템이 v24.0 미만의 레거시 엔드포인트를 호출하고 있을 가능성이 높습니다. 최신 Advantage+ 전용 엔드포인트로의 마이그레이션이 필요합니다. 개발팀에 마이그레이션 티켓을 자동으로 생성해 드릴까요?"

### 추가: CAPI 연동 상태 자동 진단
챗봇은 캠페인 생성 시 고객의 **전환 API(CAPI) 연동 여부를 필수적으로 체크**해야 한다:
- 브라우저 픽셀만 사용 중인 경우 → 데이터 누수 위험 경고 알림 발행
- CAPI 미연동 시 기여 전환수 약 24% 손실 가능성을 자동 진단하여 연동 강력 권고

## 학술 연구 기반 RAG 시스템 설계 원칙

### ARAG(Agentic RAG) 프레임워크 — 월마트 연구소
- 정적인 규칙 기반의 데이터 검색을 넘어, AI 에이전트가 실시간으로 변화하는 사용자의 세밀한 선호도와 긴 히스토리를 자율적으로 탐색하고 추천하는 차세대 RAG 방향성 제시

### 생성형 네이티브 광고 연구 (2026)
- LLM이 생성하는 답변 내에 자연스럽게 광고를 융합하는 '생성형 네이티브 광고(Generated Native Ads)'
- RAG 시스템이 사용자의 의도를 파악하여 문맥과 완벽히 일치하는 광고를 노출할 때 **수용성 극대화**

### 핵심 역량
- SaaS의 챗봇은 ARAG 구조를 벤치마킹하여, 사용자의 질문에 대해:
  1. 자율적으로 과거 6개월 치 크리에이티브 데이터 스캔
  2. 메타의 최신 알고리즘 백서 지식을 결합
  3. 근본적 해법을 논증하는 **마케팅 부사장(VP of Marketing) 급의 지능**을 발휘
