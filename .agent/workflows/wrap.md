# /wrap - 작업 마무리 다이제스트

세션을 마무리하며 오늘의 작업을 정리합니다. 아래 3단계를 순서대로 실행하세요.

---

## 1단계: 오늘의 작업 요약

`git diff --stat` 과 `git diff --name-only` 결과를 분석하여 다음을 정리합니다:

- **변경된 파일 목록**: 파일 경로와 변경 유형 (신규/수정/삭제)
- **핵심 로직 변경사항**: 각 파일에서 어떤 로직이 왜 변경되었는지 1-2줄 요약
- **해결된 이슈/버그**: 이번 세션에서 해결한 문제들
- **미해결 사항**: 작업 중 발견했지만 아직 처리하지 못한 것들

아래 형식으로 출력합니다:

```
## 작업 요약 (YYYY-MM-DD)

### 변경 파일 (N개)
| 파일 | 변경 유형 | 설명 |
|------|----------|------|
| path/to/file | 수정 | 변경 내용 요약 |

### 핵심 변경사항
- [기능/버그/리팩토링] 변경 내용 설명

### 해결된 이슈
- 이슈 설명

### 미해결 사항
- (없으면 "없음")
```

> **참고**: 커밋되지 않은 변경사항이 없으면, 가장 최근 커밋들의 diff를 분석합니다.
> `git log --oneline -10`으로 최근 커밋을 확인하고, 오늘 날짜 커밋들의 diff를 요약합니다.

---

## 2단계: 규칙/메모리 업데이트

이번 세션에서 배운 내용을 영구 기록에 반영합니다:

1. **에러 패턴**: 반복될 수 있는 에러나 함정을 발견했다면 기록
2. **프로젝트 규칙**: 새로운 컨벤션이나 주의사항이 있다면 업데이트 제안
3. **의존성/환경 변화**: 새로 설치한 패키지나 환경 설정 변경사항 기록

업데이트할 내용이 있으면 사용자에게 확인 후 반영합니다.
업데이트할 내용이 없으면 "규칙 업데이트 불필요" 로 표시합니다.

---

## 3단계: 다음 작업 To-Do 리스트

내일 바로 시작할 수 있도록 구체적인 To-Do를 생성합니다. 
> **중요**: 하루에 여러 번의 세션이 진행되어도 To-Do는 세션별로 쪼개지 않고, 파일의 **가장 맨 하단에 하나로 통합(Consolidate)** 하여 작성합니다. (만약 기존에 To-Do가 있다면 그걸 읽어온 뒤 합쳐서 덮어씁니다.)

### 전날 미완료 To-Do 이관 (필수)

오늘 날짜의 wrap 로그를 **처음 생성(세션 1)**할 때, 아래 절차를 반드시 먼저 수행합니다:

1. **전날 로그 탐색**: `docs/wrap-logs/` 디렉토리에서 오늘 이전 가장 최근 날짜의 로그 파일을 찾습니다.
   ```bash
   ls docs/wrap-logs/*.md | sort -r | head -5
   ```
2. **미완료 항목 추출**: 전날 로그의 `## 통합: 다음 작업 To-Do` 섹션에서 **체크되지 않은 항목(`- [ ]`)**만 추출합니다. 완료된 항목(`- [x]`)은 제외합니다.
3. **이관 병합**: 추출된 미완료 항목을 오늘의 To-Do에 **우선순위를 유지한 채** 병합합니다.
   - 전날 P0 → 오늘 P0 (최상단)
   - 전날 P1 → 오늘 P1
   - 전날 P2 → 오늘 P2
4. **이관 표시**: 이관된 항목에는 `(이관)` 태그를 붙여 신규 항목과 구분합니다.
5. **중복 제거**: 이번 세션에서 이미 해결한 이슈가 전날 To-Do에 있으면, 해당 항목은 이관하지 않고 `해결된 이슈`에 기록합니다.

> **세션 2 이후**: 같은 날 내 추가 세션에서는 이관 탐색을 다시 수행하지 않습니다. 이미 오늘 세션 1에서 이관이 완료되었기 때문에, 오늘 로그 내 기존 To-Do와 통합하면 됩니다.

### To-Do 형식

```
## 통합: 다음 작업 To-Do

### 우선순위 높음 (P0)
- [ ] (이관) 전날에서 넘어온 미완료 작업 (관련 파일: path/to/file)
- [ ] 이번 세션에서 새로 발견된 작업

### 우선순위 보통 (P1)
- [ ] 구체적인 작업 내용

### 나중에 (P2)
- [ ] 구체적인 작업 내용
```

- 각 To-Do에는 시작점이 되는 파일 경로를 포함
- 모호한 표현 대신 바로 실행 가능한 액션 아이템으로 작성
- 미해결 사항이 있으면 P0에 포함
- **(이관)** 태그가 붙은 항목은 전날에서 넘어온 미완료 작업임을 명시

---

## 출력 형식

위 3단계 결과를 하나의 깔끔한 마크다운으로 통합하여 출력합니다.
추가로 `docs/wrap-logs/` 디렉토리에 `YYYY-MM-DD.md` 파일로 저장합니다 (디렉토리 없으면 생성).

### 세션 번호 및 To-Do 통합 규칙
- 해당 날짜의 wrap 로그 파일이 이미 존재하면, 기존 내용 아래에 `## 세션 N` 헤더를 추가하여 1단계(작업 요약)와 2단계(규칙 업데이트)를 이어씁니다.
- **단, 3단계(To-Do)** 의 경우, 파일 내부에 존재하는 기존 To-Do를 읽어와 이번 세션의 To-Do 내용과 통합한 후 **파일의 가장 마지막에 하나의 통합된 형태**로 덮어씁니다. 중간에 산재된 여러 개의 To-Do 블록이 생기지 않도록 관리하세요.
- 없으면 파일을 새로 생성하고 `## 세션 1`로 시작하여 1~3단계를 순서대로 작성합니다.
