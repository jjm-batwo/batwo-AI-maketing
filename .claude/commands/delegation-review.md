# 위임 품질 사후 검토

현재 세션에서 수행된 에이전트 위임의 품질을 검토합니다.

## 검토 항목

### 1. Intent Gate 준수
- 모든 요청에 대해 `[Intent: ...]` 발화가 있었는가?
- 카테고리 분류가 적절했는가? (Trivial을 Open-ended로 과잉 분류하지 않았는가?)

### 2. 위임 프로토콜 준수
- 6-Section Prompt (TASK/EXPECTED OUTCOME/REQUIRED TOOLS/MUST DO/MUST NOT DO/CONTEXT)가 포함되었는가?
- `[Delegation]` 사전 선언이 있었는가?
- 레이어 매칭이 정확했는가? (Domain 작업을 designer에게 위임하지 않았는가?)

### 3. 증거 기반 완료
- `[Evidence]` 블록으로 완료를 보고했는가?
- tsc/vitest/build 결과가 실제로 첨부되었는가?
- 증거 없이 "완료" 선언한 경우가 있는가?

### 4. 실패 복구
- 3-Strike Rule이 적용된 경우가 있었는가?
- 실패 시 적절한 전문 에이전트에게 에스컬레이션했는가?
- 동일 에러에 대해 같은 접근을 반복하지 않았는가?

### 5. Wisdom 전달
- 서브에이전트 결과에서 새로운 패턴/주의사항을 추출했는가?
- MEMORY.md에 신규 발견 사항을 기록했는가?
- 후속 위임의 CONTEXT에 이전 학습을 포함했는가?

## 출력 형식

```markdown
## 위임 품질 리뷰 — [날짜]

### 요약
- 총 위임 횟수: N회
- Intent 발화율: N/N (N%)
- 6-Section 준수율: N/N (N%)
- Evidence 첨부율: N/N (N%)

### 잘한 점
- ...

### 개선 필요
- ...

### 다음 세션 권장사항
- ...
```

세션 내 대화 히스토리를 분석하여 위 항목들을 평가하고 보고서를 출력하세요.
