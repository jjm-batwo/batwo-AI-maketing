# /full-audit - 전체 최적화 감사

프로젝트의 모든 audit 커맨드를 순차 실행하여 통합 최적화 리포트를 생성합니다.

---

## 실행 대상

| # | 감사 | 설명 |
|---|------|------|
| 1 | `/bundle-audit` | 번들 크기, 트리 쉐이킹, 동적 임포트 |
| 2 | `/cache-audit` | ISR/캐싱 전략, 중복 호출, 쿼리 효율성 |
| 3 | `/rendering-audit` | 렌더링 전략 (SSG/ISR/RSC/PPR) |
| 4 | `/image-audit` | 이미지 최적화 (LCP/CLS/Bandwidth) |

---

## 워크플로우

### Step 1: 소개

```markdown
## 전체 최적화 감사

다음 4가지 감사를 순차 실행합니다:
1. 번들 최적화 감사
2. 캐싱 최적화 감사
3. 렌더링 전략 감사
4. 이미지 최적화 감사

감사 시작...
```

### Step 2: 순차 실행

각 감사를 순서대로 실행합니다. 각 감사는 해당 커맨드의 `.claude/commands/<name>-audit.md`에 정의된 절차를 따릅니다:

1. **bundle-audit** — `.claude/commands/bundle-audit.md`의 5단계 분석 실행
2. **cache-audit** — `.claude/commands/cache-audit.md`의 5단계 분석 실행
3. **rendering-audit** — `.claude/commands/rendering-audit.md`의 분석 실행
4. **image-audit** — `.claude/commands/image-audit.md`의 4단계 진단 실행

각 감사 완료 시 진행 상황을 표시합니다:

```markdown
### [1/4] 번들 최적화 감사 완료
- P0 이슈: N건
- P1 이슈: N건
- P2 이슈: N건

[다음 감사로 이동...]
```

### Step 3: 통합 리포트

모든 감사 완료 후 통합 요약을 생성합니다:

```markdown
## 전체 최적화 감사 리포트 (YYYY-MM-DD)

### 종합 요약

| 감사 항목 | P0 (즉시) | P1 (권장) | P2 (선택) | 총 이슈 |
|-----------|-----------|-----------|-----------|---------|
| 번들 최적화 | N | N | N | N |
| 캐싱 최적화 | N | N | N | N |
| 렌더링 전략 | N | N | N | N |
| 이미지 최적화 | N | N | N | N |
| **합계** | **N** | **N** | **N** | **N** |

### P0 이슈 모아보기 (즉시 조치 필요)

| # | 영역 | 대상 | 문제 | 개선 방법 |
|---|------|------|------|-----------|
| 1 | 번들/캐시/렌더링/이미지 | 파일 경로 | 문제 설명 | 개선 제안 |

### P1 이슈 모아보기 (권장)

| # | 영역 | 대상 | 문제 | 개선 방법 |
|---|------|------|------|-----------|
| 1 | 영역 | 파일 경로 | 문제 설명 | 개선 제안 |
```

### Step 4: 상세 리포트 참조

통합 요약 아래에 각 감사의 상세 리포트를 섹션별로 포함합니다:

```markdown
---

## 상세: 번들 최적화
(bundle-audit 리포트 전문)

## 상세: 캐싱 최적화
(cache-audit 리포트 전문)

## 상세: 렌더링 전략
(rendering-audit 리포트 전문)

## 상세: 이미지 최적화
(image-audit 리포트 전문)
```
