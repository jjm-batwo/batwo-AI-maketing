#### 핵심 변경사항
- **[버그/리팩토링] 캠페인 조회 Meta API Rate Limit(ErrorCode 17) 해결**: 캠페인이 많을 때 하위 아이템을 동시에 가져오며 발생하던 500 서버 에러를 해결하기 위해, `mapWithConcurrency`를 이용해 API 요청 배치를 제한(3\~5)하고, 한도 초과 시 서버가 터지지 않고 빈 배열을 반환하는 Fallback을 구축했습니다.
- **[기능/UI] 메타(Meta) UI 디자인 원복 및 개선**: 광고 세트, 광고 탭의 테이블 구조를 최근 디자인 요구상태에 맞춰 상태 제어 토글, 개별/전체 선택 체크박스, 그리고 텍스트 호버 등 메타 스타일로 온전히 복구했습니다.
- **[리팩토링] 뷰 상태 관리 (Zustand)**: `activeTab`과 선택된 아이템들의 ID를 기반으로 화면 이동 및 리스트 필터링이 가능하도록 개선하여, 탐색 중 UI가 증발하거나 텅 비는 오류를 바로잡았습니다.

#### 해결된 이슈
- "광고 세트가 없다고 나옴", 탭 이동 및 드릴다운 시 UI가 갑자기 이전 상태로 깡통이 되는 버그 수정.
- 캠페인 화면 우측 상단 "AI 챗봇" 진입점 버튼이 디자인에서 누락되었던 문제를 복구.

#### 미해결 사항
- 프로젝트 내 남아있는 앱 리뷰/목업 레코딩용 설정 파일들(Untracked/Deleted 상태인 파일들)을 정리만 남겨두고 아직 삭제 커밋하지 않음.

### 2단계: 스킬 관리 (`/manage-skills`) & 규칙 업데이트

**1. 스킬 적용 상황 (추가 스킬 생성 불필요)** 
작성 및 수정된 컴포넌트, API 경로는 기존에 설정된 `verify-ui-components` 및 `verify-token-encryption` 스킬 등에 의해 올바르게 커버리지 안에 포함됩니다. 검사 도메인 변경이나 새 컨벤션 도입이 없는 리팩토링 및 버그픽스 목적이었으므로 기존 Verify 스킬들을 그대로 쓰는 결정을 내렸습니다.

**2. 에러 패턴 모음 - Meta API Rate Limits 주의사항**
Meta 마케팅 API 연속 호출 시 `User request limit reached (17)` 발생 가능성이 큽니다. 하위 객체 (Ads/AdSets)를 Loop/Promise.all로 대량 패치할 때는 무조건 `mapWithConcurrency` 같은 유틸리티로 동시성을 3~5 정도로 강력하게 제한해야 합니다.
