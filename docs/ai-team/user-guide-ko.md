## 시작하기

### AI 개발팀이란?

바투 AI 개발팀은 **Claude 서브에이전트**를 기반으로 한 자동화 개발팀입니다. 복잡한 코딩 없이 한국어 명령만으로 시스템을 운영하고 개선할 수 있습니다.

### 팀 구성

| 역할 | 담당 업무 | 자동 실행 |
|------|----------|----------|
| **총괄 PM** | 모든 요청 접수, 작업 분배 | 항상 |
| **품질 관리자** | 테스트 실행, 품질 검증 | 코드 변경 후 |
| **보안 책임자** | 보안 검사, 취약점 탐지 | API/인증 변경 시 |
| **분석 담당** | 오류 분석, 성능 모니터링 | 에러 발생 시 |
| **설계 담당** | 아키텍처 검토 | 새 기능 요청 시 |
| **개발 담당** | 기능 구현 | 구현 요청 시 |
| **문서 담당** | 변경사항 기록 | 모든 작업 완료 후 |

### 핵심 원칙

- 🔴🟢🔵 **TDD 개발**: 테스트 먼저 → 구현 → 개선
- 🏗️ **클린 아키텍처**: 계층별 분리로 안정적인 코드
- 📋 **계획 수립**: 모든 기능은 계획서 승인 후 진행

## 기능 요청하기

### 새 기능 요청 방법

1. `/기능요청` 명령어 입력
2. PM Agent가 기능 설명을 요청합니다
3. 원하는 기능을 한국어로 설명합니다

**예시:**
```
사용자: /기능요청
PM Agent: 📝 새 기능을 설명해주세요.
사용자: 캠페인 목록에서 날짜별 필터링이 필요해요
```

### 자동 계획 수립

PM Agent가 자동으로 Feature Planner를 실행하여 계획서를 작성합니다:

```
📋 기능: 캠페인 날짜별 필터링

🏗️ 클린 아키텍처 분석:
  - domain: DateRange 값 객체 활용
  - application: FilterCampaignsUseCase 생성
  - infrastructure: Prisma 쿼리 수정
  - presentation: DatePicker 컴포넌트 추가

🔴🟢🔵 TDD 계획:
  1. RED: 필터링 유스케이스 테스트 작성
  2. GREEN: 최소 구현
  3. REFACTOR: 코드 정리

예상 작업: 3-4시간
계획 파일: docs/plans/PLAN_campaign-date-filter.md

진행하시겠습니까? (예/아니오)
```

### 계획 승인

- **"예"**: 기능 구현 시작
- **"아니오"**: 계획 수정 또는 취소

## 버그 신고하기

### 버그 신고 방법

1. `/버그신고` 명령어 입력
2. 문제 상황을 설명합니다
3. 가능하면 재현 방법도 알려주세요

**예시:**
```
사용자: /버그신고
PM Agent: 🐛 어떤 문제가 발생했나요?
사용자: 캠페인 생성 후 목록에 바로 안 나타나요. 새로고침하면 보여요.
```

### 자동 처리

- GitHub Issue가 자동 생성됩니다
- QA Agent가 문제를 분석합니다
- 수정 작업이 자동으로 진행됩니다
- 완료 시 확인 요청 알림을 받습니다

## 승인이 필요한 작업

다음 작업은 **사용자 승인 후** 진행됩니다:

| 작업 유형 | 이유 | 확인 사항 |
|----------|------|----------|
| 데이터베이스 변경 | 데이터 구조 영향 | 기존 데이터 안전 |
| API 변경 | 외부 연동 영향 | 호환성 유지 |
| 새 라이브러리 | 보안/호환성 | 필요성 검토 |
| 보안 변경 | 인증/권한 영향 | 보안 강화 |
| 프로덕션 배포 | 실서비스 영향 | 테스트 완료 |

### 승인 방법

```
PM Agent:
⚠️ 승인 필요: 데이터베이스 스키마 변경
───────────────────────────
제목: User 테이블에 role 컬럼 추가
설명: 사용자 권한 관리를 위한 role 필드 추가
변경 파일: prisma/schema.prisma

승인하시겠습니까? (/승인 또는 /거부)
```

- `/승인`: 작업 진행
- `/거부`: 작업 취소 (사유 입력 가능)

## 자주 묻는 질문

### Q: 코딩을 몰라도 사용할 수 있나요?

**A: 네, 가능합니다!** 모든 명령어는 한국어로 되어 있고, AI 에이전트가 기술적인 부분을 모두 처리합니다. 원하는 기능을 설명하기만 하면 됩니다.

### Q: 잘못된 기능이 적용되면 어떻게 하나요?

**A:** `/롤백` 명령어로 이전 버전으로 돌아갈 수 있습니다. 모든 변경은 Git으로 관리되어 언제든 복구 가능합니다.

### Q: 얼마나 자주 확인해야 하나요?

**A:**
- **일일 보고서**: 매일 오전 자동 생성
- **주간 보고서**: 매주 월요일 자동 생성
- **긴급 알림**: 문제 발생 시 즉시 알림

필요할 때만 확인하시면 됩니다.

### Q: 승인 요청은 꼭 해야 하나요?

**A: 네.** 데이터베이스나 보안 관련 변경은 실수로 인한 문제를 방지하기 위해 반드시 승인이 필요합니다. 간단한 UI 수정은 자동으로 진행됩니다.

### Q: 기능 요청 시 얼마나 자세히 설명해야 하나요?

**A:** 원하는 결과만 설명하시면 됩니다. 기술적인 구현 방법은 AI 개발팀이 알아서 결정합니다.

**좋은 예시:**
- "캠페인 목록에서 날짜로 필터링하고 싶어요"
- "보고서를 PDF로 다운로드하고 싶어요"
- "로딩이 너무 느려서 빠르게 해주세요"

**너무 기술적인 예시 (불필요):**
- "React Query의 캐시 시간을 5분으로 설정해주세요"
- "Prisma에서 include 대신 select를 사용해주세요"

### Q: 작업 진행 상황을 실시간으로 볼 수 있나요?

**A: 네.** `/진행상황` 명령어로 현재 진행 중인 모든 작업을 확인할 수 있습니다. GitHub Issue에도 자동으로 진행 상황이 코멘트됩니다.
